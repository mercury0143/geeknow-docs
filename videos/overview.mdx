# 视频系列

视频能力主要分为平台统一任务格式和 OpenAI Videos 兼容格式两套入口；两者最终都会落到异步任务体系中，再通过任务查询或内容代理获取结果。

## 路由清单

| Method | Path | 说明 |
| --- | --- | --- |
| `POST` | `/v1/video/generations` | 统一视频生成入口 |
| `GET` | `/v1/video/generations/{task_id}` | 统一任务查询接口 |
| `POST` | `/v1/videos` | OpenAI Videos 兼容创建接口 |
| `GET` | `/v1/videos/{task_id}` | OpenAI Videos 兼容查询接口 |
| `POST` | `/v1/videos/{video_id}/remix` | 对已有视频任务发起 remix |
| `GET` | `/v1/videos/{task_id}/content` | 代理下载视频内容，支持 TokenAuth 或 UserAuth |
| `POST` | `/kling/v1/videos/text2video` | Kling 官方文生视频 |
| `POST` | `/kling/v1/videos/image2video` | Kling 官方图生视频 |
| `GET` | `/kling/v1/videos/text2video/{task_id}` | Kling 文生视频查询 |
| `GET` | `/kling/v1/videos/image2video/{task_id}` | Kling 图生视频查询 |
| `POST` | `/jimeng/` | 即梦官方格式映射入口 |
| `POST` | `/sora/v1/characters` | Sora 角色创建，当前仅允许 KK 渠道 |

## 统一入口说明

- `/v1/video/generations` 与 `/v1/videos` 都会分发到具体视频渠道。
- Remix 请求会先根据 `video_id` 回查原始任务，再锁定到原渠道继续处理。
- 公开返回给客户端的任务 ID 是平台生成的 `public task id`，不会直接暴露上游真实任务 ID。

<Info>
视频接口默认是异步任务。提交成功通常先返回 `queued` 或 `in_progress` 状态，再通过查询接口拿到 `video_url` 或代理下载地址。
</Info>
