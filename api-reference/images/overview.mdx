# 图像系列

图像能力按接口和模型分别组织。每个模型页面都独立说明请求路径、模型名、参数、尺寸表和返回结构；接入时可以直接进入目标模型页面，不需要先阅读其它模型文档。

  <Card title="模型支持矩阵" icon="table-cells" href="/api-reference/images/model-matrix">
    按模型家族查看图像接口支持范围、参考图输入方式、比例和尺寸约束。
  </Card>
  <Card title="Gemini 图像生成" icon="image" href="/api-reference/images/gemini-image/generation">
    `POST /v1beta/models/{model}:generateContent`，统一覆盖 `gemini-3-pro-image-preview`、`gemini-2.5-flash-image-preview`、`gemini-3.1-flash-image-preview`。
  </Card>
  <Card title="gpt-image-2" icon="image-plus" href="/api-reference/images/gpt-image-2/generation">
    `POST /v1/images/generations`，使用 `gpt-image-2` 生成图像。
  </Card>
  <Card title="gpt-image-2-pro" icon="sparkles" href="/api-reference/images/gpt-image-2-pro/generation">
    `POST /v1/images/generations`，使用 `gpt-image-2-pro` 生成高分辨率图像。
  </Card>
  <Card title="gpt-image-2-vip" icon="sparkles" href="/api-reference/images/gpt-image-2-vip/generation">
    `POST /v1/images/generations`，使用 `gpt-image-2-vip` 的 `1K`、`2K`、`4K` 尺寸表。
  </Card>
  <Card title="图像异步生成" icon="hourglass" href="/api-reference/images/gpt-image-async/generation">
    `POST /v1/images/generations/async`，支持 GPT Image、Doubao Seedream 和 Gemini 图像模型异步提交。
  </Card>
  <Card title="图像任务查询" icon="list-checks" href="/api-reference/images/gpt-image-async/query">
    `GET /v1/images/generations/async/{taskId}`，查询异步生图任务状态、进度和结果。
  </Card>
  <Card title="Doubao Seedream" icon="image-upscale" href="/api-reference/images/doubao-seedream/generation">
    `POST /v1/images/generations`，支持 Seedream 文生图与 `image` 数组图生图。
  </Card>
  <Card title="OpenAI Images 兼容" icon="wand-sparkles" href="/api-reference/images/gpt-image-1/overview">
    基于 `POST /v1/images/generations`、`POST /v1/images/edits`、`POST /v1/edits` 的兼容入口说明。
  </Card>
  <Card title="Midjourney" icon="wand-sparkles" href="/api-reference/images/midjourney/overview">
    `/mj/submit/*`、`/mj/task/*`、`/mj/image/{id}`，适合 imagine、blend、describe、upscale 和变体操作。
  </Card>
  <Card title="上传与结果查询" icon="upload" href="/api-reference/uploads/image-upload">
    先用 `/api/upload/presign` 上传素材，再把公网 URL 传入生图、图编和视频接口。
  </Card>
</CardGroup>

## 路由清单

| Method | Path | 说明 |
| --- | --- | --- |
| `POST` | `/v1/images/generations` | 图像生成入口 |
| `POST` | `/v1/images/generations/async` | 图像异步生成入口 |
| `GET` | `/v1/images/generations/async/{taskId}` | 图像异步任务查询 |
| `POST` | `/v1/images/edits` | OpenAI Images 兼容编辑入口 |
| `POST` | `/v1/edits` | 旧版编辑别名 |
| `POST` | `/v1/images/variations` | 路由已保留，当前未实现 |
| `POST` | `/mj/submit/imagine` | Midjourney imagine |
| `POST` | `/mj/submit/blend` | Midjourney blend |
| `POST` | `/mj/submit/describe` | Midjourney describe |
| `POST` | `/mj/submit/edits` | Midjourney 编辑相关动作 |
| `POST` | `/mj/submit/video` | Midjourney 视频相关动作 |
| `GET` | `/mj/task/{id}/fetch` | Midjourney 任务查询 |
| `GET` | `/mj/image/{id}` | Midjourney 图片代理 |
| `POST` | `/v1beta/models/{model}:generateContent` | Gemini 官方格式生图入口 |

<Warning>
`/v1/images/variations`、`/v1/files*` 和 `fine-tunes*` 这类 OpenAI 兼容路径虽然在路由中占位，但当前会返回未实现错误，不能直接用于生产。
</Warning>
