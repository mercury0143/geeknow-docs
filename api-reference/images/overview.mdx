# 图像系列

图像能力目前主要分成三类：统一图像生成/编辑入口、Midjourney 专用任务接口，以及通过 Gemini 原生协议暴露的图像生成能力。

<CardGroup cols={2}>
  <Card title="GPT-Image 家族" icon="image-plus" href="/api-reference/images/gpt-image-1/overview">
    基于 `POST /v1/images/generations`、`POST /v1/images/edits`、`POST /v1/edits` 组织的模型家族页。
  </Card>
  <Card title="Midjourney" icon="wand-sparkles" href="/api-reference/tasks/midjourney-task">
    `/mj/submit/*`、`/mj/task/*`、`/mj/image/{id}`，适合 imagine、blend、describe、upscale 和变体操作。
  </Card>
  <Card title="Gemini 图像能力" icon="image" href="/api-reference/texts/gemini-native">
    通过 `POST /v1beta/models/{model}:{action}` 调用 Gemini 原生图像相关动作。
  </Card>
  <Card title="上传与结果查询" icon="upload" href="/api-reference/uploads/image-upload">
    先用 `/api/upload/presign` 上传素材，再把公网 URL 传入生图、图编和视频接口。
  </Card>
</CardGroup>

## 路由清单

| Method | Path | 说明 |
| --- | --- | --- |
| `POST` | `/v1/images/generations` | OpenAI Images 兼容生图入口 |
| `POST` | `/v1/images/edits` | OpenAI Images 兼容编辑入口 |
| `POST` | `/v1/edits` | 旧版编辑别名 |
| `POST` | `/v1/images/variations` | 路由已保留，当前未实现 |
| `POST` | `/mj/submit/imagine` | Midjourney imagine |
| `POST` | `/mj/submit/blend` | Midjourney blend |
| `POST` | `/mj/submit/describe` | Midjourney describe |
| `POST` | `/mj/submit/edits` | Midjourney 编辑相关动作 |
| `POST` | `/mj/submit/video` | Midjourney 视频相关动作 |
| `GET` | `/mj/task/{id}/fetch` | Midjourney 任务查询 |
| `GET` | `/mj/image/{id}` | Midjourney 图片代理 |
| `POST` | `/v1beta/models/{model}:{action}` | Gemini 原生图像相关动作 |

<Warning>
`/v1/images/variations`、`/v1/files*` 和 `fine-tunes*` 这类 OpenAI 兼容路径虽然在路由中占位，但当前会返回未实现错误，不能直接用于生产。
</Warning>
