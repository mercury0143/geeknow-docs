# OpenAI 音频兼容接口

这组三个音频路由都挂在 `/v1/audio/*` 下，由同一条 relay 链路处理，但请求体和响应体按具体子接口区分。

## 认证

```http
Authorization: Bearer YOUR_API_KEY
```

## 语音转文字

```http
POST /v1/audio/transcriptions
Content-Type: multipart/form-data
```

<ParamField body="file" type="file" required>
待转录的音频文件。常见格式为 `mp3`、`mp4`、`mpeg`、`m4a`、`wav`、`webm`。
</ParamField>

<ParamField body="model" type="string" required>
转录模型名称，常见值为 `whisper-1`。
</ParamField>

<ParamField body="language" type="string">
音频语言代码，例如 `zh`、`en`、`ja`。传入后通常能提升识别稳定性。
</ParamField>

<ParamField body="response_format" type="string">
响应格式，常见值为 `json`、`text`、`srt`、`verbose_json`、`vtt`。
</ParamField>

```bash
curl -X POST https://api.example.com/v1/audio/transcriptions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F file="@meeting.mp3" \
  -F model="whisper-1" \
  -F language="zh"
```

响应示例：

```json
{
  "text": "你好，欢迎使用这套语音转写接口。"
}
```

## 语音翻译

```http
POST /v1/audio/translations
Content-Type: multipart/form-data
```

这个接口和转录接口使用相同的文件上传方式，但输出目标是英文文本。

```bash
curl -X POST https://api.example.com/v1/audio/translations \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F file="@chinese_audio.mp3" \
  -F model="whisper-1"
```

响应示例：

```json
{
  "text": "Hello, welcome to this speech interface."
}
```

## 文字转语音

```http
POST /v1/audio/speech
Content-Type: application/json
```

<ParamField body="model" type="string" required>
TTS 模型，例如 `tts-1`、`tts-1-hd`。
</ParamField>

<ParamField body="input" type="string" required>
要合成的文本。
</ParamField>

<ParamField body="voice" type="string" required>
语音角色，例如 `alloy`、`echo`、`fable`、`nova`。
</ParamField>

<ParamField body="speed" type="number">
语速倍率。这个字段使用指针类型承载，显式传 `0` 与不传的语义不同，因此建议始终传入明确值。
</ParamField>

<ParamField body="response_format" type="string">
输出格式，例如 `mp3`、`wav`、`aac`、`flac`。
</ParamField>

```bash
curl -X POST https://api.example.com/v1/audio/speech \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tts-1",
    "input": "今天天气很好，适合出去散步。",
    "voice": "alloy",
    "speed": 1.0
  }' \
  --output speech.mp3
```

响应为音频二进制流，`Content-Type` 由 `response_format` 决定。

## 常见错误

| 状态码 | 说明 |
| --- | --- |
| `400` | 字段缺失、文件格式不支持、文本过长 |
| `401` | 认证失败 |
| `402` | 额度不足 |
| `413` | 请求体或上传文件过大 |
| `429` | 请求频率超限 |
| `500` | 服务端错误 |
